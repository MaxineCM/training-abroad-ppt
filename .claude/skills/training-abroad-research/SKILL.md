# 外派行前培训 — 国别信息检索与文档生成

## 概述

本 Skill 基于《课程基本信息及开发指南》和《章节大纲》，针对用户指定的国家，自动检索官方权威的最新信息（2025年1月1日之后发布），并按章节大纲结构生成一份 Word 文档（.docx），保存至项目 `Output/` 目录下。

## 核心原则

1. **权威优先**：仅引用官方渠道（政府官网、使领馆、国际组织、权威财经机构），拒绝非官方来源
2. **时效优先**：只使用 2025-01-01 之后发布的信息，找不到就标注「未获取到2025年1月1日之后的官方信息」
3. **溯源必备**：每条信息必须标注完整来源 URL 和发布日期
4. **宁缺毋滥**：找不到权威信息绝不编造，标记「待核实」
5. **全程自动**：触发后全自动执行，中间不询问用户（仅在连续搜索无果时暂停告知缺失信息）。搜索完成后自动进行自检，自检通过后直接生成文档，无需用户确认

## 工作流

### Step 1：加载参考文件

开始前，先读取以下参考文件以理解任务框架：

```
- references/课程基本信息及开发指南.md  — 课程框架、国别替换规范、版本矩阵
- references/章节大纲.md                — 6章详细大纲结构
- references/检索信息提示词.md          — 搜索标准、输出格式、示例
```

### Step 2：拆解搜索任务

根据《章节大纲》，为 6 个章节逐一列出需要检索的国别信息模块：

| 章节 | 需检索的国别信息 |
|------|-----------------|
| 第一章 | 历史快览、政经格局、经济脉搏、宗教底色、社会文化、双边关系 |
| 第二章 | 签证类型与流程、疫苗接种要求 |
| 第三章 | 住房、银行、交通、实用App、医疗体系、紧急电话、安全风险 |
| 第四章 | 宗教影响、文化禁忌、社交礼仪、商务沟通风格 |
| 第五章 | 商务法律框架（六大领域）、主要商务风险（七大类） |
| 第六章 | 利益相关方、本地商业特点、经营风险 |

### Step 3：逐章执行搜索（Tavily MCP）

全部搜索使用 **Tavily MCP** 工具链完成，无需手动 curl 或解析 HTML。

#### 3.1 Tavily MCP 工具速查

| 工具 | 用途 | 典型场景 |
|------|------|----------|
| `tavily_search` | 通用网页搜索 | 按关键词搜索，支持时间范围、域名筛选、搜索深度 |
| `tavily_extract` | 提取指定 URL 内容 | 对搜索结果中的权威 URL 获取正文（markdown/text） |
| `tavily_crawl` | 从起始 URL 爬取 | 系统性抓取政府官网的政策文件、签证页等 |
| `tavily_map` | 获取网站 URL 结构 | 了解官网结构后定向搜索 |
| `tavily_research` | 综合深度研究 | 复杂主题（如商务法律框架）的一站式研究 |

#### 3.2 两层搜索策略

**第1层：`tavily_search` — 关键词搜索（首选入口）**

```
对每个信息模块，使用 `references/检索信息提示词.md` 第3.2节的推荐搜索关键词，
调用 tavily_search 执行搜索。

推荐参数：
  - search_depth: "advanced"      # 对关键信息使用深度搜索
  - time_range: "year"            # 只返回最近一年的内容（时效性要求）
  - max_results: 10               # 每次搜索 10 条结果
  - include_domains: []           # 可限定只搜官网（如 .gov.sg）
  - country: "目标国英文名"        # 提升目标国本地结果权重
```

**第2层：`tavily_extract` — 权威来源提取**

```
对 tavily_search 返回的权威 URL（.gov / .org / 官方机构），
调用 tavily_extract 获取完整正文：

推荐参数：
  - extract_depth: "advanced"     # 表格、嵌入内容较多的页面用 advanced
  - format: "markdown"            # 保留结构化信息
  - include_images: false         # 不需要图片（减少噪音）
```

> **世界银行数据**：仍可通过 `Bash(curl)` 直接调用 World Bank API 获取结构化经济数据（GDP、人口等），公式为：
> `curl -s "https://api.worldbank.org/v2/country/[ISO]/indicator/[指标]?format=json&per_page=3"`
>
> 常用指标：`NY.GDP.MKTP.KD.ZG`(GDP增长率) `SP.POP.TOTL`(人口) `SL.UEM.TOTL.ZS`(失业率)
> ISO 代码速查：SG=新加坡 MY=马来西亚 DE=德国 BR=巴西 VN=越南 ID=印尼 TH=泰国 IN=印度 SA=沙特 AE=阿联酋

#### 3.3 补充策略

当 `tavily_search` 返回的权威官方信息不足时：
- **tavily_map** → 先了解目标官网的 URL 结构
- **tavily_crawl** → 对官网进行定向爬取（设置 `select_paths` 限定路径，`max_depth: 2`）
- **tavily_research** → 对复杂、跨领域的主题（如第5章商务法律框架）使用综合深度研究

#### 3.4 搜索执行规范

对每条信息：
- ✅ 来自官方渠道 + 2025年1月1日后发布 → 采纳，标注来源 URL + 日期
- ❌ 找不到官方最新信息 → 标记 `【未获取到2025年1月1日之后的官方信息】`

关键信息（如 GDP 数据、政策法规）必须交叉验证 2 个以上来源。

搜索关键词格式：`国家名 + 信息主题 + 限定词`，参考 `references/检索信息提示词.md` 中第3.2节各章节的推荐搜索词。

#### 3.5 混合模式处理

默认全自动执行。当某章关键信息连续搜索无果时，暂停并告知用户：
- 已搜索的渠道
- 缺失的具体信息
- 询问是否跳过该模块或提供替代来源

### Step 4：整理信息

将所有搜索结果按《章节大纲》结构编排，遵循 `references/检索信息提示词.md` 中的输出格式规范：

- 每条信息标注：`【来源：完整URL；发布时间：YYYY年MM月DD日】`
- 多源信息可整合在同一段落，各来源独立标注
- 保留《章节大纲》中原有的通用内容（如心态建设、安全场景应对等）
- 国别替换模块用搜索结果填充

### Step 4.5：自检（必须执行，无需用户确认）

整理完所有章节信息后，**必须**对照以下三份参考文件逐章自检：

**自检清单：**

| 检查维度 | 对照标准 | 不合格处理 |
|----------|----------|-----------|
| **章节完整性** | 《章节大纲》6章所有标题/子标题是否均已覆盖 | 缺漏→补充搜索 |
| **国别替换覆盖** | 《课程基本信息及开发指南》"国别信息替换规范"表格中的模块是否均已填充 | 缺漏→补充搜索 |
| **通用内容保留** | "通用内容稳定模块"列表中的模块是否已标注「通用内容」 | 缺漏→补充标注 |
| **信息权威性** | 每条国别信息是否来自 .gov /.org / 国际组织 / 权威机构 | 非权威来源→补充搜索或降级标注 |
| **信息时效性** | 每条国别信息发布时间是否在 2025-01-01 之后 | 过时→重新搜索或标记「未获取到最新官方信息」 |
| **来源标注** | 每条国别信息是否标注了完整 URL + 发布时间 | 缺漏→补充标注 |

**执行方式**：自检全程不询问用户。发现问题自动补充搜索/修正。全部通过后进入 Step 5。

### Step 5：生成 Word 文档（自检通过后自动执行）

自检通过后，**立即自动生成文档，无需用户确认**。使用 `docx` Skill 生成 .docx 文件，存放路径：

```
Output/[国家名]检索信息结果.docx
```

**文档格式要求**：

- 标题层级与《章节大纲》严格一致（第一章 → 1.1 → 四级标题）
- 表格内容保留表格格式
- 来源标注使用小号字体（比正文小1-2号）
- 未获取到信息的模块保留原标题，下方标注「未获取到2025年1月1日之后的官方信息」
- 通用内容模块（如 2.4 心态建设、3.4 安全场景）保留原文框架，标注「通用内容」

### Step 6：输出摘要

文档生成完成后告知用户：
- 文件路径及大小
- 各章节信息获取情况摘要（表格：✅ 已获取 / ⚠️ 部分缺失 / ❌ 无官方信息）
- 提示哪些模块需要人工补充

## 注意事项

- 本 Skill 默认在当前项目 `D:\AIcraete\Training-abroad-PPT` 下运行
- `Output/` 目录首次生成前可能不存在，需自动创建
- 生成的是 Word 文档（.docx），后续可配合 `pptx` Skill 转换为 PPT
- 批量处理多国时，每个国家独立执行一次完整流程
