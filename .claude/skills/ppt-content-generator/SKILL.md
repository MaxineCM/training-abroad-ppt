# 外派行前培训 — PPT内容提示词文档生成器

## 概述

本 Skill 读取 `Output/检索信息结果/[国家名]检索信息结果.docx`，严格按照 `references/第一章PPT大纲样例` 的格式，生成分章节的PPT内容提示词文档，保存至 `Output/PPT内容文件/[国家名]出国培训课程文档/` 文件夹。

## 核心原则

1. **严格对齐模板**：每个章节文档的格式、结构、粒度完全对齐样例模板（精确到每一页幻灯片）
2. **内容不编造**：所有内容来自检索信息文档，不添加虚构信息，缺失的标注「待补充」
3. **表格优先**：能用表格呈现的内容优先用表格，减少纯文字段落
4. **全程自动**：触发后全流程不询问用户，一次生成全部7个文件
5. **框架不偏离**：内容组织和时长分配严格遵循《章节大纲》和《课程基本信息及开发指南》

## 工作流

### Step 1：读取输入

读取以下文件：

```
Output/检索信息结果/[国家名]检索信息结果.docx  — 国别检索信息（核心内容来源）
references/第一章PPT大纲样例                      — 文档格式模板（已内嵌在 references 目录）
Basic Outline/章节大纲                           — 内容框架 + 时长分配
Basic Outline/课程基本信息及开发指南              — 课程核心目标、通用/国别模块区分
```

**⚠️ 关键**：`references/第一章PPT大纲样例` 是本 skill 的核心格式参考。此文件取自项目 `Basic Outline/第一章PPT大纲样例.docx`，需先解包读取其完整内容，理解其：
- 每页幻灯片的结构（标题 → 内容要点 → 表格 → 视觉建议）
- 封面页格式（章标题 + 副标题 + 讲师信息 + 配图建议）
- 小节总结页格式（核心特征 + 商业启示对照）
- 字体层级、标注方式（✅ / ⚠️ / 视觉建议 / 配图建议）

### Step 2：规划页面分配

根据《章节大纲》中各章节的标准时长，按约 **2分钟/页** 的比例确定每章PPT页数：

| 章节 | 时长 | 建议页数 |
|------|------|----------|
| 第一章 国情概况 | 30分钟 | 14-16页 |
| 第二章 签证与行前准备 | 35分钟 | 17-19页 |
| 第三章 生活安顿与安全常驻 | 40分钟 | 20-22页 |
| 第四章 文化洞察与社交礼仪 | 30分钟 | 14-16页 |
| 第五章 通用商务规则 | 30分钟 | 14-16页 |
| 第六章 高效履职与团队合作 | 30分钟 | 14-16页 |
| 附录 | — | 3-5页 |

实际页数根据检索信息文档中该章节的内容量微调（内容多的章可适当增加，内容少的章可减少）。

### Step 3：逐章生成内容文档

为每章创建独立 docx 文件，严格遵循以下每页结构：

```
每页幻灯片格式（对齐模板）：
┌─────────────────────────────────────────┐
│ 幻灯片 N：标题                            │
│                                         │
│ 【内容要点】                              │
│ · 要点1                                  │
│ · 要点2                                  │
│                                         │
│ 【数据表格】（如适用）                      │
│ | 维度 | 核心内容 | 商业启示 |             │
│                                         │
│ 【视觉建议】配图建议：xxx                    │
│ 【讲师备注】补充说明（如适用）               │
└─────────────────────────────────────────┘
```

**生成顺序**：
1. 封面页（章标题 + 副标题 + 国家名 + 讲师信息留白 + 配图建议）
2. 核心知识点总览页
3. 逐节内容页（每页一张幻灯片）
4. 小节总结页（核心特征 → 商业启示对照表）

### Step 4：生成附录文档

附录内容包含：
- 各章节信息获取情况汇总表
- 仍需人工补充的模块清单
- 紧急联系速查表
- 参考来源汇总

### Step 5：组织文件夹并输出

创建文件夹结构并保存所有文档：

```
Output/PPT内容文件/[国家名]出国培训课程文档/
├── [国家名]第一章课程内容.docx
├── [国家名]第二章课程内容.docx
├── [国家名]第三章课程内容.docx
├── [国家名]第四章课程内容.docx
├── [国家名]第五章课程内容.docx
├── [国家名]第六章课程内容.docx
└── [国家名]附录.docx
```

### Step 6：输出摘要

生成完成后告知用户：
- 文件夹完整路径
- 7个文件的列表及各自页数
- 总体完成情况
- **提示用户审查内容并提出调整意见**

### Step 7：响应用户调整（版本管理）

**7.1 调整原则**

用户审查后提出调整意见时：
- **只修改用户明确提到的部分**，不改用户未提及的内容
- **不偏离** Basic Outline 的整体框架和核心主题

**7.2 版本命名规则**

每次调整后**不覆盖原文件**，而是另存为新版本：

| 版本 | 文件名示例 | 说明 |
|------|-----------|------|
| 初始 | `日本第一章课程内容.docx` | 首次生成的无版本标识文件 |
| 第1次调整 | `日本第一章课程内容V1.docx` | 用户第一次提调整意见后生成 |
| 第2次调整 | `日本第一章课程内容V2.docx` | 用户第二次提调整意见后生成 |
| 第N次调整 | `日本第一章课程内容VN.docx` | 依此类推 |

> 注意：只有用户明确要求调整的章节才生成新版本文件，其他章节保持不变。

**7.3 调整后询问**

每次完成调整后**必须询问用户**：
> "还有其他需要调整的地方吗？"

- 如果用户说"有" → 继续执行调整（再次递增版本号）
- 如果用户说"没有了" / "没有" / "没了" → 进入 Step 7.4

**7.4 最终整理**

当用户确认没有更多调整后，告知用户：
> "好的，我来整理课程文件夹。我会把之前调整产生的旧版本文件（V1、V2...）删除，每个章节只保留最新版本，确保课程文档包整洁干净。确认吗？"

用户确认后：
1. 删除该章节所有旧版本文件（如 V1、V2...）
2. 保留最终版本（即最高版本号的文件）
3. 告知用户整理完成，skill 调用结束

**整理示例**：
```
调整前：日本第一章课程内容.docx / 日本第一章课程内容V1.docx / 日本第一章课程内容V2.docx（最新）
↓ 用户确认整理
整理后：日本第一章课程内容V2.docx（最新，唯一保留）
```
